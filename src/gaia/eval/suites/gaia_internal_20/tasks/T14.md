# Task

The fixture at ``fixtures/T14.patch`` seeds a path-traversal vulnerability
into ``src/gaia/coder/tools/file.py``. The function ``read_file(path)`` is
supposed to refuse paths outside the configured sandbox, but the seeded
implementation uses a plain ``startswith`` check against the sandbox path —
which is bypassed by ``../`` components that happen to resolve into the
sandbox by coincidence, and by ``sandbox/../sandbox-evil/secrets.txt``.

A seeded test in ``tests/unit/test_t14_seeded.py`` asserts:

1. ``read_file("../../etc/passwd")`` raises a ``SandboxEscapeError``.
2. ``read_file("sandbox/../sandbox-evil/secrets.txt")`` raises a
   ``SandboxEscapeError``.
3. Normal reads inside the sandbox still succeed.

## Required work

1. Replace the ``startswith`` check with a proper ``Path.resolve() +
   is_relative_to(sandbox_root.resolve())`` guard (Python 3.9+).
2. Keep the existing allow-path tests passing.
3. Add a test for the symlink-escape case (create a symlink inside the
   sandbox that points to ``/tmp`` and confirm we still refuse it).

## Acceptance

- All three ``test_t14_seeded`` cases pass.
- The PR body cites [CLAUDE.md "No Silent Fallbacks" rule](../../../../../CLAUDE.md#no-silent-fallbacks--fail-loudly)
  and notes that the fix raises a typed error rather than returning None.
- Diff ≤ 30 LoC.
