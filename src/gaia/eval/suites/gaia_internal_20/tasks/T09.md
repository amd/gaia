# Task

The fixture at ``fixtures/T09.patch`` seeds an over-broad deny rule in
``src/gaia/coder/tools/cli.py`` — the string match ``"rm "`` rejects
``"rm-safe --dry-run"`` because the denylist uses ``substring in`` instead
of word-boundary matching.

A seeded test in ``tests/unit/test_t09_seeded.py`` asserts that
``run(["rm-safe", "--dry-run"])`` should NOT raise ``ShellDeniedError``.

Fix the deny check so it matches whole argv[0] tokens only (not substrings
in the joined command string).

## Required work

1. Locate the seeded denylist in ``src/gaia/coder/tools/cli.py``.
2. Replace substring matching with token-level matching — compare against
   ``argv[0]`` (or the basename of ``argv[0]``) rather than the joined string.
3. Keep the real deny list working — ``run(["rm", "-rf", "/"])`` must STILL
   be denied. Add a test confirming the true-positive still trips.

## Acceptance

- ``pytest tests/unit/test_t09_seeded.py -x`` passes (both false-positive
  and true-positive sub-tests).
- Diff ≤ 25 LoC.
- The PR body explicitly notes the security implication of the fix
  ("widened allow-list would have been a regression; this is token-level,
  same-or-stricter behaviour").

## Budget

Diff ≤ 25 LoC, ≤ 15 min wall clock.
