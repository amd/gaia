# Task

Add a new tool method ``screenshot_url(url: str, *, viewport: str = "1280x720",
wait_ms: int = 500) -> str`` to ``ScreenshotToolsMixin`` in
[`src/gaia/agents/tools/screenshot_tools.py`](../../../../agents/tools/screenshot_tools.py).

The tool should:

- Launch a headless browser (use `playwright` since it's already a project
  dependency), navigate to the URL, wait `wait_ms` ms, capture a full-page
  screenshot, and return the absolute path to the saved PNG.
- Save the PNG under `~/.gaia/screenshots/` with a `yyyy-mm-dd-hhmmss-<hash>.png`
  filename so multiple captures don't collide.
- Be registered via the existing `@tool` decorator with a docstring that the
  agent can show the user.

## Acceptance

- ``screenshot_url`` is importable from ``gaia.agents.tools.screenshot_tools``.
- ``tests/unit/test_screenshot_tools.py`` has at least one new test that mocks
  the playwright call and asserts the returned path lives under
  ``~/.gaia/screenshots/``. (Mock via `monkeypatch`; do not hit the network.)
- ``python util/lint.py --all`` has no new errors.
- The PR body notes that this mirrors the existing `screenshot_region` tool.
