# Task

Create a new GAIA agent called ``weather`` that exposes a single tool:

```python
@tool
def get_weather(location: str, *, units: str = "metric") -> dict:
    """Return today's forecast for `location` as {'temp': ..., 'summary': ...}."""
```

You may stub the actual weather fetch behind a ``_fetch`` helper that tests
monkey-patch — no live API keys, no real network calls from the unit test.

## Required work

1. Create `src/gaia/agents/weather/agent.py` with a `WeatherAgent(Agent)` class
   that inherits from the base `Agent` and registers `get_weather` via
   `@tool`. Model default is `Qwen3-0.6B-GGUF` (match other lightweight agents).
2. Create `src/gaia/agents/weather/__init__.py` exporting `WeatherAgent`.
3. Wire the agent into `src/gaia/agents/registry.py` — register under key
   `weather` so `load_agent("weather")` succeeds.
4. Add `tests/unit/test_agent_weather.py` with at least two tests:
   one for the tool-call round-trip (mocked LLM), one for registry discovery.

## Acceptance

- ``from gaia.agents.weather.agent import WeatherAgent`` works.
- ``load_agent("weather")`` returns a ``WeatherAgent`` instance.
- ``pytest tests/unit/test_agent_weather.py -x`` passes.
- ``python util/lint.py --all`` passes.
- Diff ≤ 300 LoC.
