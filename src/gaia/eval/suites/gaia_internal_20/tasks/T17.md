# Task

The fixture at ``fixtures/T17.patch`` seeds a flaky test ``test_cache_ttl``
in ``tests/unit/test_t17_flaky.py``. The test:

- Starts a background timer that expires after a random 50-150 ms.
- Asserts the cache entry still exists after a 100 ms sleep.

Flake-rate measured ~15% over 20 runs.

## Required work

1. **Diagnose the root cause** — write one short paragraph in the PR body
   explaining WHY the test is flaky (race between timer expiry and the
   100 ms sleep).
2. **Propose a fix that is either**:
   - a proper mock of the timer (best), OR
   - a `@pytest.mark.flaky(reruns=3)` quarantine with a TODO comment citing
     this task ID, if the proper fix is out of scope for the 40-LoC budget.
3. Run the test 10 times in a row (`pytest --count=10` via `pytest-repeat`)
   and confirm zero failures.

## Acceptance

- ``pytest tests/unit/test_t17_flaky.py -x --count=10`` exits 0.
- PR body has a "Root cause" paragraph.
- If quarantined, a TODO with task ID is present in the test file.

## Budget

Diff ≤ 40 LoC, ≤ 25 min wall clock.
