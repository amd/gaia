## 1. Overview

GAIA Pipeline Orchestration is an objective-driven project execution system built on top of the GAIA PipelineEngine. It manages complex, multi-step projects by breaking them into discrete **objectives** with dependencies, then executing them in order (sequential) or in parallel groups (levels).

### Architecture

```
Objectives YAML -> ProjectOrchestrator -> PipelineEngine -> Results
                        |                        |
                  ProjectSupervisor      CircuitBreaker
                        |
                  SupervisorRegistry -> GitSupervisor
                        |
                  HookRegistry -> [6 hooks]
                        |
                  REST API + SSE Stream -> UI Dashboard
```

### GAIA UI - Home Screen

The GAIA Agent UI provides the central hub for managing orchestrator tasks:

![GAIA UI Home](screenshots/orch-01-gaia-ui-home.png)
*Figure 1: GAIA Agent UI - The starting point for running orchestrator tasks. Navigate to "Run Pipeline" in the sidebar to access the orchestrator dashboard.*

### Key Features

| Feature | Description |
|---------|-------------|
| **Dependency-Aware Scheduling** | Automatic topological ordering, cycle detection, cascade failure propagation |
| **Parallel Execution** | Level-based parallelism with configurable concurrency limits |
| **Git Worktree Isolation** | Each objective runs in its own git branch and worktree |
| **Supervisor Governance** | Health scoring, pause/resume, abort on repeated failures |
| **Conflict Detection** | File-level overlap detection between parallel objectives |
| **Automatic Rollback** | Git reset on failed objectives when supervisor aborts |
| **Hook System** | 6 built-in hooks for git automation and objective lifecycle |
| **Real-Time Visibility** | REST API + SSE streaming for live monitoring |
| **UI Dashboard** | Web-based interface for watching and controlling execution |

## 2. Quick Start

### Step 1: Install GAIA

```bash
uv venv && uv pip install -e ".[dev,ui]"
```

### Step 2: Create Your Objectives File

Create `.gaia/objectives.yaml` in your project root:

```yaml
project_id: my-project
name: "My GAIA Project"
objectives:
  - objective_id: obj-001
    title: "Initialize Project"
    description: "Set up project structure"
    status: queued
    dependencies: []
    priority: 1
    phase: PLANNING
  - objective_id: obj-002
    title: "Build Core Module"
    description: "Implement core functionality"
    status: queued
    dependencies: [obj-001]
    priority: 2
    phase: DEVELOPMENT
```

### Step 3: Start the UI Server

```bash
gaia chat --ui
```

The server starts on `http://localhost:4200`. On startup, it auto-detects your `.gaia/objectives.yaml` and initializes the orchestrator.

### Step 4: Run the Orchestrator

**Via API (recommended):**

```bash
curl -X POST http://localhost:4200/api/v1/orchestrator/run
```

**Via Python:**

```python
from gaia.orchestration import ProjectOrchestrator, OrchestratorConfig

config = OrchestratorConfig(
    objectives_path=".gaia/objectives.yaml",
    enable_parallel_execution=True,
    enable_supervisor=True,
)
orchestrator = ProjectOrchestrator(config=config)
await orchestrator.run()
```

### Step 5: Monitor Progress

```bash
# Get current state
curl http://localhost:4200/api/v1/orchestrator/state

# Stream real-time events
curl -N http://localhost:4200/api/v1/orchestrator/stream
```

## 4. Objectives YAML Reference

### Complete Schema

```yaml
project_id: <string>                  # Auto-generated UUID if omitted
name: <string>                        # Human-readable project name
objectives:
  - objective_id: <string>            # Auto-generated if omitted
    title: <string>                   # Human-readable title (required)
    description: <string>             # Becomes pipeline user_goal
    status: <enum>                    # queued | in_progress | completed | blocked | cancelled
    dependencies: [<string>]          # List of objective_ids
    priority: <int>                   # Lower = higher priority (default: 5)
    phase: <string>                   # PLANNING | DEVELOPMENT | QUALITY | DECISION
    pipeline_config:                  # Passed to PipelineEngine
      template: <string>              # Template name (default: generic)
      quality_threshold: <float>      # 0.0-1.0 (default: 0.90)
      max_iterations: <int>           # Max iterations (default: 10)
      concurrent_loops: <int>         # Parallel loops (default: 1)
    artifacts:                        # Auto-populated on completion
      - name: <string>
        artifact_type: <string>
        url_or_path: <string>
        metadata: <dict>
    created_at: <ISO timestamp>       # Auto-generated
    updated_at: <ISO timestamp>       # Auto-updated
    error_message: <string|null>      # Set on failure
metadata: <dict>                      # Arbitrary project-level metadata
```

### Status Values

| Status | Description | Can Transition To |
|--------|-------------|-------------------|
| `queued` | Waiting to be executed | in_progress, blocked, cancelled |
| `in_progress` | Currently executing | completed, blocked, cancelled |
| `completed` | Successfully finished | (terminal) |
| `blocked` | Failed or dependency not met | queued (retry), cancelled |
| `cancelled` | User-aborted | (terminal) |

**Two-Step Transition System:**

All status transitions require an intermediate `in_progress` step: `queued -> in_progress -> target`. This prevents race conditions where a status could skip the in_progress phase.

## 5. Configuration

### OrchestratorConfig

```python
from gaia.orchestration import OrchestratorConfig

config = OrchestratorConfig(
    # Required
    objectives_path=".gaia/objectives.yaml",

    # Git
    auto_commit=False,               # Auto-git-commit after each objective

    # Execution
    dry_run=False,                   # Preview without executing
    enable_parallel_execution=True,  # Use parallel (level-based) mode
    max_parallel_objectives=10,      # Max concurrent objectives
    enable_evaluation=False,         # LLM-based post-execution eval
    max_cycle_iterations=100,        # Max dispatch cycles

    # Monitoring
    enable_nexus=True,               # Event logging via NexusService

    # Governance
    enable_supervisor=True,          # Enable ProjectSupervisor
    supervisor_config=SupervisorConfig(
        consecutive_failure_threshold=3,
        max_consecutive_failures=5,
        quality_window=5,
    ),

    # Git Supervisor
    enable_git_supervisor=True,      # CircuitBreaker-protected git ops

    # Rollback
    enable_rollback=True,            # Rollback on ABORT/PAUSE

    # Hooks
    serialize_hooks=True,            # Serialize hook execution
)
```

### SupervisorConfig

```python
from gaia.orchestration import SupervisorConfig

config = SupervisorConfig(
    consecutive_failure_threshold=3,   # Pause after 3 consecutive failures
    max_consecutive_failures=5,        # Abort after 5 consecutive failures
    quality_window=5,                  # Rolling window for quality trend
    quality_decline_threshold=0.10,    # Slope triggering REMEDIATE
    health_success_rate_weight=0.40,   # Weight in composite health
    health_quality_weight=0.30,
    health_dependency_weight=0.30,
    health_minimum_score=0.50,
    max_remediation_depth=2,           # Max remediation cycles
    min_trend_slope=0.05,              # Min slope for REMEDIATE
)
```

### Starting via Python

```python
import asyncio
from gaia.orchestration import ProjectOrchestrator, OrchestratorConfig

async def main():
    config = OrchestratorConfig(
        objectives_path=".gaia/objectives.yaml",
        enable_parallel_execution=True,
        enable_supervisor=True,
        enable_git_supervisor=True,
        enable_rollback=True,
    )

    orchestrator = ProjectOrchestrator(config=config)
    state = await orchestrator.run()

    print(f"Completed {state.objectives_processed} objectives")
    print(f"Failed {state.objectives_failed} objectives")
    print(f"Total cycles: {state.cycle_count}")

asyncio.run(main())
```

### Sequential vs Parallel Mode

| Feature | Sequential (`False`) | Parallel (`True`) |
|---------|---------------------|-------------------|
| Execution order | One at a time, topological order | Level-based parallel groups |
| Worktrees | Not created | One per objective |
| Conflict detection | N/A | Pairwise file comparison |
| Concurrency | 1 | Configurable (max_parallel_objectives) |
| Speed | Slower, predictable | Faster, depends on dependencies |

## 8. Real-Time SSE Streaming

Connect to the SSE stream for live event updates:

```bash
curl -N http://localhost:4200/api/v1/orchestrator/stream
```

### Event Types

| Event | When Fired | Example Payload |
|-------|-----------|-----------------|
| `orchestrator_start` | Dispatch loop begins | `{"type":"orchestrator_start","pipeline_id":"orchestrator-demo"}` |
| `objective_start` | Before executing objective | `{"type":"objective_start","objective_id":"obj-001","phase":"PLANNING"}` |
| `objective_complete` | Objective succeeded | `{"type":"objective_complete","objective_id":"obj-001","quality_score":0.92}` |
| `objective_failed` | Objective failed | `{"type":"objective_failed","objective_id":"obj-002","error":"Pipeline error"}` |
| `cycle_complete` | End of dispatch cycle | `{"type":"cycle_complete","cycle_count":1}` |
| `phase_complete` | All objectives in phase done | `{"type":"phase_complete","phase":"PLANNING"}` |
| `orchestrator_paused` | Pause triggered | `{"type":"orchestrator_paused","reason":"Too many failures"}` |
| `orchestrator_resumed` | Resume triggered | `{"type":"orchestrator_resumed"}` |
| `orchestrator_complete` | All done | `{"type":"orchestrator_complete","state":{...}}` |
| `orchestrator_error` | Unhandled exception | `{"type":"orchestrator_error","error":"Connection refused"}` |

### SSE Format

Events arrive in SSE format:
```
data: {"type":"objective_start","objective_id":"obj-001",...}

data: {"type":"objective_complete","objective_id":"obj-001",...}

: keepalive
```

A `: keepalive` ping is sent every 30 seconds to prevent proxy timeouts.

### SSE Architecture

The `OrchestratorSSEBridge` class manages fan-out event distribution:
- Each client gets an `asyncio.Queue(maxsize=1000)` 
- Events are broadcast to all subscribers via `bridge.broadcast()`
- Clients disconnect cleanly via `CancelledError` handling
- Anti-buffering headers prevent proxy delays

## 12. Hooks System

Hooks are event-driven callbacks that fire at specific lifecycle points.

### Hook Event Timeline

**Sequential Mode:**
```
ORCHESTRATOR_START
  -> Loop:
     -> OBJECTIVE_START (before dispatch)
     -> Pipeline execution
     -> OBJECTIVE_COMPLETE or OBJECTIVE_FAILED (after dispatch)
     -> Supervisor evaluation
     -> CYCLE_COMPLETE
     -> PHASE_COMPLETE (if phase is done)
  -> ORCHESTRATOR_COMPLETE
```

### Available Hooks

| Hook | Listens To | Purpose |
|------|-----------|---------|
| `ObjectiveUpdateHook` | OBJECTIVE_COMPLETE | Atomically saves objectives.yaml |
| `TaskSpawnHook` | OBJECTIVE_FAILED | Creates remediation objectives |
| `GitBranchHook` | OBJECTIVE_START | Creates feature branch |
| `GitCommitHook` | OBJECTIVE_COMPLETE | Commits objectives.yaml |
| `GitPRHook` | ORCHESTRATOR_COMPLETE | Creates PR with summary |
| `GitRollbackHook` | OBJECTIVE_FAILED | Rolls back branch |

### Registering Hooks

```python
from gaia.orchestration import (
    ProjectOrchestrator, OrchestratorConfig,
    GitBranchHook, GitCommitHook, GitPRHook,
    ObjectiveUpdateHook, TaskSpawnHook,
)

orchestrator = ProjectOrchestrator(config=config)

# Register hooks
orchestrator.hook_registry.register(
    event_name="OBJECTIVE_START",
    handler=GitBranchHook(git_supervisor=orchestrator.git_supervisor),
    priority=HookPriority.HIGH,
)

orchestrator.hook_registry.register(
    event_name="OBJECTIVE_COMPLETE",
    handler=GitCommitHook(git_supervisor=orchestrator.git_supervisor),
    priority=HookPriority.NORMAL,
)

# Run
await orchestrator.run()
```

## 13. Supervisor Governance

The ProjectSupervisor monitors execution health and issues verdicts:

### Verdicts

| Verdict | Trigger | Orchestrator Response |
|---------|---------|----------------------|
| **CONTINUE** | All checks passed | Continue to next objective |
| **PAUSE** | >= 3 consecutive failures (default) | Set `state.paused`, wait for resume |
| **REMEDIATE** | Quality trend declining | Log warning, increment remediation depth |
| **ABORT** | >= 5 consecutive failures (default) | Break dispatch loop, trigger rollback |

### Health Score

Composite score (0.0-1.0) computed from:

```
success_rate = successful / total      (weight: 40%)
quality_trend = normalized OLS slope   (weight: 30%)
dependency_health = 1 - blocked/total  (weight: 30%)

composite = success_rate * 0.40 + quality_trend * 0.30 + dependency_health * 0.30
```

### Pause/Resume

```bash
# Pause with reason
curl -X POST http://localhost:4200/api/v1/orchestrator/pause \
  -H "Content-Type: application/json" \
  -d '{"reason": "Manual pause for review"}'

# Resume
curl -X POST http://localhost:4200/api/v1/orchestrator/resume
```

## 15. Troubleshooting

### Common Issues

| Problem | Cause | Solution |
|---------|-------|----------|
| **Orchestrator not found** | Server not started or `.gaia/objectives.yaml` missing | Run `gaia chat --ui` and verify YAML exists |
| **409 Conflict on /run** | Already running or flag not reset | Check if orchestrator is running; restart server if flag stuck |
| **404 Objective not found** | Wrong objective_id | List objectives with `GET /api/v1/orchestrator/objectives` |
| **Circular dependency error** | Cycle in objective dependencies | Remove the cycle from your YAML |
| **All objectives blocked** | Root objective failed or cycle detected | Fix the root cause, then reset status to `queued` |
| **Hook halts pipeline** | `halt_pipeline=True` returned by hook | Check hook logs for the reason |
| **Worktree creation fails** | Git not initialized or branch exists | Run `git init` and clean stale branches |
| **Circuit breaker open** | 3+ consecutive git failures | Wait 60s for recovery, or fix git issues |

  "offset": 0
