The Problem

How do you know your AI agent actually works? Manual testing doesn't scale, unit tests can't catch hallucinations, and users find the bugs first.

🔍

Invisible Failures

Tool-calling failures, hallucinated facts, and format errors slip through code review and unit tests.

📉

Quality Regressions

A prompt tweak that fixes one scenario silently breaks three others. No regression detection means surprises in production.

⌛

Slow Feedback Loop

Manual testing takes hours per release. Engineers avoid touching agent code because the blast radius is unknown.

The Solution

An automated eval harness that uses Claude Code as both user simulator and judge to validate agent quality against real-world scenarios.

Architecture

How It Works

  • Scenario YAML defines persona, setup documents, multi-turn conversation, and ground truth facts
  • Claude subprocess (claude -p) acts as both user simulator and judge in a single pass
  • MCP protocol connects to the Agent UI backend for real end-to-end testing
  • Scorecard engine aggregates results with deterministic weighted scoring
  • Fix mode auto-invokes Claude Code to diagnose and fix failures
# Scenario YAML format
id: cross_turn_file_recall
category: context_retention
persona: data_analyst

setup:
  index_documents:
    - path: eval/corpus/docs/report.html

turns:
  - turn: 1
    objective: Index document
    user_message: "Read report.html"
    success_criteria: "Agent reads file"

  - turn: 2
    objective: Ask about content
    user_message: "What was the Q3 revenue?"
    ground_truth:
      facts: ["$4.2M"]
      source_files: ["report.html"]

7-Dimension Scoring

Every scenario is scored across 7 dimensions. Scores are deterministically recomputed from dimension weights—no LLM arithmetic.

DimensionWeightWhat It Measures
Correctness 25% Factual accuracy against ground truth
Tool Selection 20% Chose the right tools, avoided unnecessary ones
Context Retention 20% Maintained context across conversation turns
Completeness 15% Addressed all parts of the user's question
Efficiency 10% Concise response, no unnecessary tool calls
Personality 5% Matched the persona's communication style
Error Recovery 5% Graceful handling of missing data or tool failures

PASS score ≥ 6.0 AND all turns correctness ≥ 4    FAIL score < 6.0 OR any turn correctness < 4

Fix Mode & Regression Detection

Auto-Fix Loop

When a scenario fails, the --fix flag invokes Claude Code to diagnose the root cause and apply a fix, then re-evaluates automatically.

$ gaia eval agent --fix \
    --max-fix-iterations 3 \
    --target-pass-rate 0.90

Iteration 1: 45/54 passed (83%)
  Fixing: hallucination_resistance...
  Applied: prompt guard in agent.py
Iteration 2: 49/54 passed (91%)
Target pass rate reached!

Extensibility

Third-party developers can extend the eval framework without modifying GAIA source code.

Custom Scenarios

Drop YAML files in your own directory and they're auto-discovered alongside built-in scenarios.

$ gaia eval agent \
    --scenario-dir ~/my-project/eval/

Custom Corpus

Add your own documents and manifests for domain-specific testing.

$ gaia eval agent \
    --corpus-dir ~/my-project/corpus/

Scenario Tags

Tag scenarios with metadata for selective runs.

# In scenario YAML:
tags: [healthcare, critical]

# Run only tagged scenarios:
$ gaia eval agent --tag healthcare

CI/CD Integration

JUnit XML output for GitHub Actions, Jenkins, and other CI systems.

$ gaia eval agent \
    --output-format junit

Getting Started

# Install GAIA
$ pip install gaia-ai

# Start Agent UI backend
$ gaia chat --ui

# Run all eval scenarios
$ gaia eval agent

# Run a specific category
$ gaia eval agent --category rag_quality

# Architecture audit (free)
$ gaia eval agent --audit-only

# Save as baseline for regressions
$ gaia eval agent --save-baseline

# Compare after changes
$ gaia eval agent --compare baseline.json

# Auto-fix failures
$ gaia eval agent --fix
